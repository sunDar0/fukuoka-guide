# 가족 동선 설계

큐레이션된 명소를 가족이 실제로 따라갈 수 있는 일자별 동선으로 엮고, `planner.js`의 일정 템플릿 재작성안을 만드는 방법을 정의한다.

## 왜 동선 설계가 따로 필요한가

명소를 잘 골라도 동선이 무리면 여행이 고생이 된다. 특히 유아·노부모 동반은 "몇 군데 보느냐"가 아니라 "얼마나 여유 있느냐"가 만족을 결정한다. 좌표·이동 수단·체력 페이스를 함께 풀어야 실제로 굴러가는 일정이 나온다.

## planner.js itineraryTemplates 스키마

일정은 `window.itineraryTemplates[nights]` 객체 딕셔너리다(nights = 숙박일수, 2~7). 3박4일은 `itineraryTemplates[3]`이고 `planner.js:119` 부근에 있다. 구조:

```js
window.itineraryTemplates[3] = {
  nights: 3,
  days: [
    {
      day: 1,
      title: '하카타 도착 & 시내 탐방',     // 한국어 day 제목
      summary: '...',                       // 한 줄 요약
      timeline: [
        { time: '11:00', icon: '✈️',
          activity: '후쿠오카 공항 도착',
          details: '입국 수속 후 ... 이동',  // 이동 수단·소요시간·주차를 여기 적는다
          childNote: '공항 내 유아 휴게실 이용 가능' }  // 유아 있을 때만 렌더됨
      ]
    },
    // day 2, 3, 4 ...
  ]
}
```

- `days` 배열 길이 = nights + 1 (3박이면 4일).
- `renderItinerary`는 `infantsCount > 0`일 때만 `childNote`를 표시한다. 노부모 배려는 별도 렌더 슬롯이 없으므로 `details`에 녹이거나 `childNote`에 함께 적는다.
- `icon`은 이모지 1개. `time`은 'HH:MM' 문자열.

## 동선 설계 원칙

### 이동 수단이 동선을 결정한다
- **렌터카**: 활동 간 이동은 자동차 소요시간 기준. `details`에 "차로 ○분, △△ 주차장(시간당 ¥□□)"을 적는다. 도심 주차난 구역(텐진·나카스)은 호텔/공영 주차장 전제로 묶고, 외곽·근교·온천은 렌터카 강점을 살린다. 공항 픽업/반납 시점을 day1/마지막 day에 명시.
- **대중교통**: 역·정류장·패스 기준. (현재 기본 템플릿이 이 방식)

### 페이스 원칙 (유아·노부모)
- 하루 핵심 명소 2~3개 + 식사·휴식. 욕심내지 않는다.
- 유아 낮잠 슬롯(보통 오후 1~3시)을 이동 시간이나 카페·호텔 휴식과 겹친다.
- 노부모용으로 앉을 곳·실내·체류 긴 명소를 오후에 배치.
- 첫날(도착)·마지막날(출국)은 이동 비중이 크니 가볍게.

### 거점 전략
- "시내 N박 + 온천 M박" 같은 구조를 지킨다. **짐 옮기는 날은 명소를 최소화**하고 체크아웃→이동→체크인을 동선 중심에 둔다.
- 같은 거점에 묵는 날은 거점 반경 안에서 묶고, 근교 당일치기는 왕복 시간을 좌표로 따져 하루 안에 무리 없는지 확인한다.

### 좌표 검증
- 인접 활동의 좌표 거리를 대략 계산해(위경도 차 × 대략 111km/도) 하루 총 이동이 과한지 본다. 렌터카 하루 누적 이동은 여유 있게 잡는다.

## 산출물 형식 (`_workspace/02_route_plan.md`)

일자별로 위 timeline 스키마대로 쓰되, **각 활동이 가리키는 data.js 명소명을 괄호로 명시**한다(경계면 검증용).

```markdown
## Day 1 — 하카타 도착 & 시내 (거점: 시내)
- 11:00 ✈️ 공항 도착 / details: 렌터카 픽업(공항점), 차로 시내 호텔 15분 / childNote: 공항 수유실
- 14:00 🛍️ 캐널시티 하카타 [data.js: 캐널시티 하카타] / details: 호텔 주차 후 도보, 차로 5분 / childNote: 실내, 유아차 가능
```

## 하지 않을 것

- `planner.js`를 직접 수정하지 않는다(integrator의 몫). 재작성안만 산출한다.
- 명소를 새로 발굴하지 않는다. 풀 조정이 필요하면 local-curator에게 요청한다.
- 시간을 분 단위로 과하게 쪼개지 않는다. 가족 여행은 버퍼가 핵심이다.
