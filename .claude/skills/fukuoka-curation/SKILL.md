# 후쿠오카 명소 큐레이션

후쿠오카 현지 가이드 관점에서 `data.js`의 `locations` 명소 풀을 여행자 조건에 맞게 골라내는 방법을 정의한다.

## 왜 큐레이션이 필요한가

58개 명소를 다 넣으면 가이드가 아니라 목록이다. 가족 구성·시즌·이동 수단이 정해지면 적합한 명소는 절반 이하로 줄어든다. 현지 가이드의 가치는 "거긴 겨울에 닫혀요", "유아차로 그 계단은 못 올라가요"처럼 **빼야 할 이유를 아는 것**이다. 빼는 판단이 넣는 판단만큼 중요하다.

## data.js locations 스키마

각 명소 객체의 필드는 고정이다. 새 필드를 만들지 않는다.

```js
{
  name: '캐널시티 하카타',        // 한국어 명소명 (고유 키 역할)
  lat: 33.5898, lng: 130.4110,   // 좌표 (동선·거리 계산에 사용)
  region: '하카타',               // regionCenters의 8개 중 하나
  description: '...',             // 한두 문장. 시즌·유아·노부모 메모를 여기 녹인다
  hours: '10:00~21:00',          // 영업시간
  admission: '무료',              // 입장료
  website: 'https://...',
  duration: 120,                  // 체류 권장(분) — 일정 시간 배분에 사용
  childFriendly: true,            // 유아 동반 적합 여부
  imageQuery: 'Canal City ...',   // Unsplash 검색어 (영어)
  nearbyHotels: [ { name, distance, price } ]  // 빈 배열 허용
}
```

region은 다음 8개만 사용한다: 하카타, 텐진/나카스, 다자이후, 후쿠오카 시내, 이토시마, 키타큐슈, 유후인, 벳푸.

## 큐레이션 필터 (조건별 판단 기준)

조건은 작업 시작 시 가정으로 한 줄 명시한다. 아래는 자주 쓰는 필터다.

### 유아(영유아) 동반
- `childFriendly: false`이거나, 계단·경사·긴 도보·좁은 좌석(야타이 등) 위주면 **제외 또는 주의**.
- 수유실·기저귀 교환대·유아차 접근성이 있으면 가점. 모르면 WebSearch로 확인하거나 "확인 필요" 표시.
- 실내·넓은 공간·짧은 체류(duration 작음)가 유리.

### 노부모(연장자) 동반
- 가파른 계단·긴 도보·장시간 서있기 위주면 감점. 벤치·엘리베이터·휠체어 접근성이 있으면 가점.
- 하루에 몰아넣지 않게, 체류 길고 앉을 곳 있는 명소를 거점 근처에 배치하도록 메모.

### 시즌 (겨울/12월)
- **야외 위주 명소**(전망대 야외, 해변, 꽃·정원 시즌)는 추위·휴장으로 감점. description에 "4~5월 등나무", "벚꽃 시즌" 같은 계절 한정 문구가 있으면 12월엔 매력이 떨어짐을 메모.
- **겨울 가점**: 일루미네이션 명소, 실내(쇼핑몰·박물관·온천), 모츠나베·딸기 같은 겨울 먹거리 연계.
- 구체적인 겨울 휴장일·이벤트 기간은 단정하지 말고 WebSearch로 확인하거나 "현지 확인 필요"로 남긴다.

### 이동 수단
- 렌터카: 주차장 유무가 중요. 주차 어려운 도심 한복판(텐진 등)은 "주차 확인 필요" 메모. 외곽·근교는 렌터카에 유리.
- 대중교통: 역·정류장 도보 거리 기준.

## 추가/제거 의사결정 규칙

1. **유지**: 조건 필터를 모두 통과 + 거점 동선에 들어맞음.
2. **제외**: 시즌·유아·노부모 중 하나라도 명확히 부적합. 사유를 반드시 단다.
3. **추가**: 현재 풀에 없지만 이 조건에 강하게 맞는 명소(예: 겨울 일루미네이션, 유아 실내 놀이시설). 추가 시 위 스키마 전 필드를 채워 제안한다. 좌표·영업시간이 불확실하면 WebSearch로 채우고, 못 채우면 "확인 필요"로 표시.
4. **경합**: 큐레이션 가치와 동선 효율이 충돌하면 삭제하지 말고 둘 다 기록해 route-planner·사용자 판단에 맡긴다.

## 산출물 형식 (`_workspace/01_curator_shortlist.md`)

지역(거점)별로 묶고, 각 명소에 판정·사유·data.js 반영 메모를 단다.

```markdown
## 거점: 하카타/텐진 (시내)
- [유지] 캐널시티 하카타 — 실내 복합몰, 겨울·유아·노부모 모두 무난. 반영: description에 "겨울 실내 코스" 한 줄 추가
- [제외] ○○ 전망대(야외) — 12월 추위 + 노출 동선. 사유: 겨울 야외 부적합
- [추가] △△ 일루미네이션 — 12월 한정 야간 명소. 좌표/시간 WebSearch 확인됨. 반영: 신규 객체(전 필드 기재)
```

## 하지 않을 것

- `data.js`를 직접 수정하지 않는다(integrator의 몫). 무엇을 어떻게 바꿀지만 산출한다.
- 불확실한 사실을 단정하지 않는다. "확인 필요"가 틀린 단정보다 낫다.
- 조건과 무관한 명소를 "유명하니까" 넣지 않는다.
